## 확인 사항

- 서버 재시작 후 로그에서 `[ZendriverDaemon] Service ready for reuse` 가 출력되는 것을 확인했다.
- 초기 실사용 로그에서 데몬 준비 중 중복 기동 레이스가 발생해 `Failed to start (timeout)` 및 `Address already in use` 로 이어지는 문제를 추가 확인했다.
- [mod_ohli24.py](/Volumes/WD/Users/Work/python/ff_dev_plugins/anime_downloader/mod_ohli24.py) 에서 데몬 시작 락을 `RLock` 으로 강화하고, 살아 있는 기존 프로세스가 있으면 새로 띄우지 않고 health/browse-ready 상태가 될 때까지 더 길게 대기하도록 보정했다.
- fetch 경로는 `browser_ready=True` 까지 기다린 뒤 데몬 요청을 보내도록 조정했고, watchdog 도 데몬 부팅 직후에는 grace 기간 동안 재시작을 보류하도록 수정했다.
- 추가로 [lib/zendriver_daemon.py](/Volumes/WD/Users/Work/python/ff_dev_plugins/anime_downloader/lib/zendriver_daemon.py) 의 HTTP 서버를 단일 스레드 `HTTPServer` 에서 `ThreadingHTTPServer` 로 변경해, 긴 `/fetch` 처리 중에도 `/health` 가 응답 가능하도록 수정했다.
- 이 변경으로 fetch 중 health 타임아웃 때문에 데몬이 죽은 것으로 오판되어 중복 기동과 `Address already in use` 가 발생하던 조건을 줄이도록 했다.
- 공용 `Zendriver` bootstrap 책임을 내부 모듈별 `plugin_load()` 에서 제거하고, 총괄 진입점 [__init__.py](/Volumes/WD/Users/Work/python/ff_dev_plugins/anime_downloader/__init__.py) 의 `plugin_load()` 에서 한 번만 실행하도록 옮겼다.
- 이에 따라 [mod_ohli24.py](/Volumes/WD/Users/Work/python/ff_dev_plugins/anime_downloader/mod_ohli24.py), [mod_anilife.py](/Volumes/WD/Users/Work/python/ff_dev_plugins/anime_downloader/mod_anilife.py) 의 중복 bootstrap 호출을 제거해 시작 로그와 책임 경계를 정리했다.
- [mod_anilife.py](/Volumes/WD/Users/Work/python/ff_dev_plugins/anime_downloader/mod_anilife.py) 의 GDM 위임 경로를 [mod_ohli24.py](/Volumes/WD/Users/Work/python/ff_dev_plugins/anime_downloader/mod_ohli24.py) 패턴과 맞춰, `prepare_extra()` 로 실제 비디오 URL과 저장 경로를 먼저 확보한 뒤 `ModuleQueue.add_download()` 를 호출하도록 수정했다.
- 이 변경으로 `save_path=None` 상태로 GDM 등록을 시도하다가 `expected str, bytes or os.PathLike object, not NoneType` 에러가 나던 문제를 막고, `anilife` 다운로드가 GDM 큐로 정상 진입할 수 있게 조정했다.
- [anime_downloader_anilife_request.html](/Volumes/WD/Users/Work/python/ff_dev_plugins/anime_downloader/templates/anime_downloader_anilife_request.html) 의 단건 다운로드 버튼 성공 조건에도 `enqueue_gdm_success` 를 추가해, GDM 위임 성공 시 프런트가 실패처럼 보이지 않도록 맞췄다.
- 검증: `python -m py_compile mod_ohli24.py mod_anilife.py lib/zendriver_daemon.py lib/zendriver_ohli24.py`
